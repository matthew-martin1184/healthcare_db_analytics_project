Project Overview and Demonstration

This project develops an interactive dashboard in Streamlit for exploratory analysis of a healthcare provider network using data from a public-domain Kaggle dataset. It implements an end-to-end ETL pipeline, built in Jupyter Notebooks using Pandas, to transform raw CSV files into a structured relational schema and load the data into an Oracle Autonomous Data Warehouse. The data is queried using SQL and visualized through the dashboard to support insights into provider activity and patient encounters.

ETL Pipeline

This pipeline was developed in Jupyter Notebooks using Python, Pandas, Numpy, and Oracledb, to do the following:

  • Extract data from Kaggle API View Kaggle Dataset
  • Convert CSV files to Pandas DataFrames View Raw CSV Files
  • Clean and transform data in preparation for loading to Oracle Database View Cleaned CSV Files
  • Insert data into live Oracle Autonomous Data Warehouse

Data Wrangling for Dashboard

This Notebook implements the following procedures to wrangle data for the analysis dashboard.

  • Load data from the Oracle database
  • Parse descriptions from a txt file using Regex View Description File
  • Create data objects for the dashboard
  • Serialize and dump the objects to a host environment for dashboard View .pkl File

Streamlit Dashboard

## The dashboard is currently in development.

This dashboard will load serialized data objects using Pickle. The data can be filtered using dropdowns to select for patient, doctor, medical procedure, or billing insights. Additional filtering can be applied to target specific relationships. These filters plot data from the objects wrangled from the result sets of the SQL queries. Visualizations will be displayed using Matplotlib.
